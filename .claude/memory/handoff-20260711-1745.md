## Active task: node-name collision fix (IN PROGRESS, blocked on disk)

### The bug
- This Claude Code session (in `/Users/james/Projects/meshtest`) was named `copilot-cli-mac` — wrong; that's Copilot's node.
- Nodes on the same computer collide on names.

### Root cause (confirmed)
`my_node()` resolves identity as `--as` > `A2ACAST_NODE` env > per-directory `.meshwire.node` file, defaulting to `_default_node_name()` = **bare hostname** (`jamess-macbook-air-2`). Nothing in the derivation includes the **harness** or **session**. So (a) every agent in one directory reads the same `.meshwire.node` → same name (Claude inherited copilot's), and (b) any two default nodes on one machine both get the hostname → collide. The nice names (`codex-cli-mac`, etc.) were all set manually via `--as`/`iam`; there is no automatic uniqueness.

### Fix decision (user chose)
**Harness-aware stable names**, and `mesh iam` can still rename. Model: default node = `{host}-{harness}`; identity resolves per-harness; a session never inherits another harness's shared file.
